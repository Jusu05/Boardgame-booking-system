# Docstring Notifications
 
Most of the report's notifications are like the following:
 
```
app.py:26:0: C0116: Missing function or method docstring (missing-function-docstring)
```
 
These notifications mean that in modules, classes and functions there are no docstring comments. In application development, a conscious decision has been made not to use docstring comments.
 
## Control Flow
 
```
app.py:186:8: R1705: Unnecessary "else" after "return", remove the "else" and de-indent the code inside it (no-else-return)
```
 
This comes from a situation where else makes the code more readable.
 
```
app.py:308:0: R0912: Too many branches (24/12) (too-many-branches)
app.py:668:0: R0912: Too many branches (15/12) (too-many-branches)
app.py:39:0: R0912: Too many branches (14/12) (too-many-branches)
```
 
These come from match statements where there are more than 12 branches. This is a conscious application design decision because the match statements control most of the business logic of the application, which gets input from one form field value.
 
```

app.py:668:0: R0911: Too many return statements (8/6) (too-many-return-statements)
```

This is caused by the same reason because all the business logic is separated into own functions.
